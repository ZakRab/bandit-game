EPSILON-GREEDY: THE CODE
success_rate = {}
for each connection_type:
    success_rate[type] = successes / attempts

if random() < 0.2:         # explore
    choice = random_choice()
else:                      # exploit
    choice = max(success_rate)

pull(choice)
update_stats(choice, outcome)
O(k)
per decision

THE PROBLEM WITH EPSILON-GREEDY
It treats all unknowns equally!
Alumni: 5 tries → 1 success (20%)
HIGH CONFIDENCE
You know it's probably bad
Different: 1 try → 1 success (100%)
HIGH UNCERTAINTY
Could be great, could be luck!
Thompson's Insight:  Be optimistic about uncertainty
KENNY — 14:00-15:00• 'Epsilon-greedy has a problem — it treats all unknowns the same'• Walk through the two cards:  - Alumni: tried 5 times, 1 success → you're pretty confident it's ~20%. Not great.  - Different: tried 1 time, 1 success → 100% rate, but that could be total luck!• The key question: should you explore 'Different' more? YES — you're uncertain about it• But epsilon-greedy doesn't care about uncertainty — it just picks randomly 20% of the time• Thompson's insight: BE OPTIMISTIC ABOUT UNCERTAINTY• If you're unsure about something, give it a chance — it might be gold• This is the intuition behind Thompson Sampling
11

THOMPSON SAMPLING: BAYESIAN APPROACH
Model each option as Beta(successes + 1, failures + 1)
THE ALGORITHM:  Sample from each distribution → Pick the highest sample
KENNY — 15:00-16:00• 'We model each option as a probability distribution using the Beta distribution'• Don't get bogged down in math — focus on the intuition:  - More data → narrower distribution → more confident  - Less data → wider distribution → more uncertain• Alumni example: tried 5 times, narrow curve, you KNOW it's around 20%• Different example: tried once, wide curve — could be anywhere from 10% to 90%!• The algorithm: take a random sample from each distribution, pick the highest• Why this works: uncertain options sometimes sample HIGH, so you naturally explore them• Confident good options stay high, so you naturally exploit them• 'The math automatically balances explore vs exploit for you'
12

THOMPSON SAMPLING: THE CODE
for each connection_type:
    α = successes + 1
    β = failures + 1
    θ ~ Beta(α, β)   # Sample

choice = argmax(θ)    # Pick highest
O(k)
per decision
O(log T)
regret — provably optimal!
USED AT
Netflix, Google, FDA

REGRET ANALYSIS
Regret = gap between optimal outcome and what you actually got
Random
O(T)
Linear — keeps making mistakes
Epsilon-Greedy
O(T^{2/3})
Sublinear — good
Thompson
O(log T)
Logarithmic — optimal!
WILSON — 22:00-22:30• Define regret simply: 'The gap between the best you COULD have done and what you actually got'• Walk through the three curves:  - Random: linear — you keep making mistakes at the same rate forever  - Epsilon-Greedy: sublinear — gets better over time but never fully optimal  - Thompson: logarithmic — flattens out fast, barely any regret after a while• 'Thompson is provably near-optimal — you literally can't do much better than O(log T)'• If you have class data: 'Your class average regret was about X per round — optimal would have been around Y'• This is why Thompson is the industry standard for production systems• Key takeaway: smarter exploration = less regret = better outcomes faster
